# 序言

企业建设 Agent 平台时，最容易低估的不是模型能力，而是系统边界。一个可以演示的 Agent 往往只需要模型、提示词和几个工具；一个能够长期运行的企业级 Agent 平台，则必须回答更多问题：任务如何被定义，工具如何被授权，数据如何被解释，运行过程如何被审计，失败以后如何恢复，成本和风险又如何被持续治理。

这本书围绕这些问题展开。它不把 Agent 写成单一框架教程，也不把 DataAgent 简化为 NL2SQL。全书更关心企业系统中的接口、状态、证据、权限、评估和组织协作：模型只是其中一层能力，数据底座、知识工程、运行时、工具注册、可观测性、部署和安全治理共同决定了 Agent 能否进入生产。

中文第一版采用“平台工程”视角组织内容。前半部分先建立 Agent 平台的基本语言，解释模型、数据、检索和 Runtime 之间的关系；中间部分以 DataAgent 为主线，把语义层、NL2SQL、Python 分析、可视化报告和评估串成可交付链路；后半部分回到可观测性、成本、部署、安全、合规和组织演进。读者可以按顺序阅读，也可以把本书当成架构评审、方案设计和问题排查时的参考手册。

本书仍处在第一版整理阶段。对于尚未获得真实材料支撑的业务案例、致谢和项目闭环，当前版本不会编造内容来填满目录；对于已经进入正文的章节，则优先保证结构、图表、引用、代码路径和网页电子书体验可检查、可复审、可迭代。

希望这本书能帮助读者把“做一个 Agent”推进到“建设一套可治理的平台能力”：不是只追求某次演示成功，而是让团队能够在真实业务、真实权限、真实数据和真实故障中持续交付。
