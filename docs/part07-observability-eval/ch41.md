## 1. Agent 成本结构分析

本节拆解 Agent 成本来源：模型输入输出 token、长上下文、工具调用、检索、代码执行、向量查询、会话回放存储、评测调用和重试成本。

重点说明 Agent 成本与普通 LLM 应用不同：一次任务可能包含多次模型调用、多轮工具调用和失败重试，因此成本应按 run、step、tenant、agent、model、tool 等维度归集。

## 2. 模型路由与动态选型

本节介绍模型路由策略：按任务类型、风险等级、上下文长度、延迟要求、成本预算和历史效果动态选择模型。

建议覆盖路由模式：规则路由、小模型预判、大模型兜底、级联调用、失败切换、按租户配额路由。要强调路由必须接入评测指标，不能只按价格选模型。

## 3. Prompt Cache 与 Prefix Cache

本节区分 Prompt Cache 与 Prefix Cache。前者偏应用层复用稳定提示词、系统指令和上下文片段；后者偏推理服务层复用相同前缀的 KV Cache。

建议说明适用场景：固定系统提示词、长工具说明、稳定 schema、常用政策文档、批量相似任务。也要说明失效策略：Prompt 版本、工具版本、权限上下文和租户隔离变化都可能导致缓存不可复用。

## 4. Semantic Cache 与结果复用

本节介绍语义缓存：对语义相似的问题复用已有答案、SQL、分析过程或中间结果。它适合 FAQ、指标解释、固定报表问答和低风险重复查询。

风险要点包括：相似但不等价的问题、权限差异、时间窗口变化、数据刷新、个性化上下文。企业场景中语义缓存必须有置信度、过期时间和权限校验。

## 5. Token 成本核算与预算控制

本节定义成本核算口径：输入 token、输出 token、缓存命中 token、重试 token、评测 token、按模型单价换算后的金额。

预算控制可以按用户、租户、Agent、场景、项目和时间窗口设置。建议讨论软限制、硬限制、告警、降级、审批和成本归因报表。

## 6. 性能、成本与质量平衡策略

本节总结工程取舍：更便宜的模型可能降低任务成功率，更激进的缓存可能带来过期答案，更短上下文可能丢失关键证据，更少重试可能降低可用性。

建议给出决策框架：先保障安全和正确性，再优化延迟，最后压降成本；对低风险高频任务优先缓存，对高风险低频任务优先准确和可审计。

## 本章小结

### 关键结论

1. Agent 成本要按任务链路归集，不能只统计单次模型调用。
2. 模型路由的目标是“合适任务用合适模型”，不是简单选择最便宜模型。
3. Prompt / Prefix / Semantic Cache 解决的问题不同，失效条件也不同。
4. 预算控制要和租户、权限、SLO、评测指标联动。
5. 成本优化必须保留质量护栏，否则会把隐性风险转移到业务侧。
